# Add a client-scoped resource

Build a new feature as a full vertical slice, matching the existing code exactly.
Copy the closest existing feature as a template â€” **compliance** (simple CRUD) or
**calendar** (has sub-objects + workflow) are the best references.

## House rules (must hold)
1. **Layering:** `router â†’ service â†’ repository â†’ db`. Routers stay thin.
2. **Repositories never commit.** The **service owns the commit**; keep multi-step writes atomic.
3. **Object-level authz:** client-scoped routes call `ClientService(db).get_client(user, client_id)`
   first â€” it returns **404 (not 403)** for an inaccessible client. Never trust the path id alone.
4. **Bounded inputs:** request schemas inherit `StrictModel` (`extra="forbid"`), cap free-text with
   `max_length` (`MAX_TEXT` / `MAX_LONG_LINE` from `app/schemas/common.py`), and cap list sizes.
5. **Bounded outputs:** list endpoints paginate via the shared `Pagination` dependency and return
   `{ items, total, page, page_size }`. Push `LIMIT`/`OFFSET` + a `func.count()` to the repository.
6. **Typed errors only:** raise from `app/core/exceptions.py`; never build HTTP responses in services.
7. **Config:** any new setting goes in `app/core/config/*.py` and `.env.example` â€” never `os.environ`.

## Steps

### 1. Model â€” `app/models/<name>.py`
- Inherit `Base` + the right mixins (`UUIDPrimaryKeyMixin`, `TimestampMixin` or `CreatedAtMixin`).
- Use the portable types from `app.models.base`: `GUID`, `TZDateTime`, `JSONColumn`, `pg_enum(...)`.
- Client-scoped rows need `client_id: Mapped[uuid.UUID] = mapped_column(GUID,
  ForeignKey("clients.id", ondelete="CASCADE"), nullable=False, index=True)`.
- Add new enums to `app/models/enums.py`. Register the model in `app/models/__init__.py`.

### 2. Migration
- `make migration m="add <name>"` then **review** the autogenerated file in `migrations/versions/`.
- Ensure both `upgrade()` and a real `downgrade()` exist. Confirm FK `ondelete`, indexes, and enum
  handling match the model. Never hand-edit already-applied migrations.

### 3. Schemas â€” `app/schemas/<name>.py`
- Request models inherit `StrictModel`; read models inherit `ORMModel` (`from_attributes=True`).
- Cap every free-text field and list. Use `EmailStr` for emails.
- Add a `<Name>ListResponse` with `items`, `total`, `page: int = 1`, `page_size: int = 20`.

### 4. Repository â€” `app/repositories/<name>_repository.py`
- Subclass `BaseRepository[<Model>]`; set `model = <Model>`.
- **Hard-filter every query by `client_id`.** Never commit here (flush only, if needed).
- `list_for_client(...)` returns `tuple[list[Model], int]` â€” a page plus a DB-side count:
  ```python
  total = self.db.scalar(select(func.count()).select_from(Model).where(*conditions))
  stmt = select(Model).where(*conditions).order_by(...).offset(offset)
  if limit is not None:
      stmt = stmt.limit(limit)
  return list(self.db.scalars(stmt).all()), int(total or 0)
  ```

### 5. Service â€” `app/services/<name>_service.py`
- Enforce scoping is done at the router; still hard-filter by `client_id`.
- Own the transaction: mutate, then `self.db.commit()`. On integrity errors, `rollback()` and raise
  `ConflictError`. Raise `NotFoundError` when a row is missing.
- List method takes `pagination: PaginationParams` and returns the `<Name>ListResponse`.
- If the change should trigger an intelligence rebuild, enqueue via `JobQueue(self.db).enqueue(...)`
  in the same transaction (see `ComplianceService`).

### 6. Router â€” `app/api/v1/routers/<name>.py`
- `router = APIRouter(prefix="/clients/{client_id}/<name>", tags=["<name>"])`.
- Use the `deps.py` aliases: `CurrentUser` / `AdminUser`, `DbSession`, `Pagination`, `StorageDep`.
- First line of each handler: `ClientService(db).get_client(user, client_id)`.
- Set `response_model` on every route, `status_code=201` on creates.
- Rate-limit paid/expensive routes: `dependencies=[Depends(RateLimit("<scope>", times=..., seconds=60))]`.
- **Register the router in `app/api/v1/api.py`** (`api_router.include_router(<name>.router)`).

### 7. Tests â€” `tests/integration/test_<name>.py`
- Use the `client`, `admin_headers`, and `make_user` fixtures from `tests/conftest.py`.
- Cover: create â†’ list (assert pagination shape) â†’ get â†’ update â†’ delete; a **403/404
  authorization** case (a non-assigned user must get 404); and validation failures (422).

## Verify
```bash
.venv/bin/python -m ruff check app tests
.venv/bin/python -m pytest tests/integration/test_<name>.py -q
```
Then run the whole suite once: `.venv/bin/python -m pytest -q`. All green before you finish.
